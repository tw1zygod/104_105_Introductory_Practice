# Практическое задание: Разработка веб-форм и применение UI-анимаций

## Цель задания

Освоить семантику интерактивных элементов для сбора данных. Спроектировать доступную (accessible) веб-форму и оживить интерфейс сайта с помощью CSS-псевдоклассов и плавных транзиций.

## Техническое задание

Ваша задача — добавить на сайт функциональный блок с формой и настроить красивые микроанимации.

### 1. Архитектура веб-формы (HTML)

Придумайте, для чего нужна форма на вашем сайте. Это может быть классическая форма «Связаться со мной», анкета «Оцените мое портфолио», форма заказа ваших услуг.

1. Добавьте контейнер `<form>`. В качестве заглушки укажите атрибуты `action="#"` и `method="POST"`.
2. Реализуйте минимум **три** различных поля ввода (`<input>`).

**!!!** Выбирайте типы полей под вашу задачу. Это могут быть текст (`text`), почта (`email`), дата (`date`), ползунок (`range`), галочки (`checkbox`) или переключатели (`radio`).


3. Добавьте в форму логичную кнопку отправки данных (`<button type="submit">`).
4. **Техническое требование (a11y):** Каждое поле ввода должно быть строго связано со своей текстовой подписью `<label>`. Используйте для этого точное совпадение атрибутов `for` (у подписи) и `id` (у поля ввода).

### 2. Уникальная компоновка формы (CSS)

Спроектируйте дизайн формы так, чтобы она органично вписывалась в ваш сайт.

1. Оберните каждую логическую пару «Подпись + Поле» в отдельный `<div>` (группу).
2. Используйте Flexbox или Grid для выстраивания элементов. Ваша форма может быть строгой вертикальной колонкой, а может быть разбита на два столбца.
3. Задайте полям ввода комфортные внутренние отступы (`padding`), подберите уникальный цвет фона, скругление углов (`border-radius`) и стиль рамок.

### 3. Управление фокусом (`:focus`)

Обеспечьте визуальную доступность интерфейса при вводе данных.

1. В CSS обратитесь к вашим полям ввода через псевдокласс `:focus` и отключите стандартную браузерную обводку (`outline: none;`).
2. Придумайте свой собственный стиль для активного поля. Это может быть смена цвета фона, появление красивой тени (`box-shadow`), утолщение рамки или изменение цвета текста. Пользователю должно быть интуитивно понятно, где сейчас находится курсор.

### 4. Визуальный отклик и плавность (`:hover` и `transition`)

Оживите все кликабельные элементы на вашем сайте (кнопку в форме, ссылки в навигации, карточки проектов).

1. Придумайте интересные эффекты при наведении мыши (`:hover`). Кнопка может становиться ярче, карточка проекта — слегка приподниматься (через тень), а ссылки — менять цвет или получать подчеркивание.
2. Задайте базовым элементам (не псевдоклассам!) свойство `transition`.
3. Настройте длительность анимации (рекомендуемый диапазон: от `0.2s` до `0.4s`), чтобы эффекты срабатывали плавно и не раздражали глаз.

## Строгие правила (Чего делать нельзя)

1. **Никаких потерянных лейблов.** Запрещено оборачивать текст просто в `<p>` или `<span>` рядом с инпутом. Только `<label>` с рабочим атрибутом `for`.
2. **Сохраняйте читаемость (Модульность).** Не пишите стили для инпутов прямо в селекторах тегов (`input { ... }`), используйте понятные классы (например, `.form-input`, `.form-group`, `.submit-btn`), чтобы код оставался масштабируемым.
3. **Следуйте принципу KISS (Keep It Simple, Stupid) в анимациях.** Анимируйте только те свойства, которые не влияют на физическую геометрию страницы (цвет фона, цвет текста, рамки, прозрачность, тени). Категорически запрещено изменять свойства `width`, `height`, `margin` или `padding` при `:hover` — это вызывает пересчет всего макета и заставляет соседние элементы "дергаться".
