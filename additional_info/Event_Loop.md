# Event Loop (Событийный цикл) и Асинхронность

Движок V8 (Call Stack и Memory Heap) умеет выполнять только синхронный код. Но как тогда работают таймеры или запросы к серверу, не блокируя весь сайт? Здесь на помощь приходит браузер (или Node.js) и механизм Event Loop.

## 1. Web APIs (Браузерное окружение)

Движок JS **не имеет** встроенного `setTimeout`, `fetch` или работы с DOM. Это всё — **Web APIs**, которые предоставляет браузер.

Они написаны на низкоуровневых языках (типа C++) и выполняются за пределами главного потока JS.

**Как это работает:**

Когда JS встречает `setTimeout`, он передает его в Web API и говорит: *"Эй, браузер, отсчитай 2 секунды, а когда закончишь — вот тебе функция (коллбэк), которую надо будет выполнить"*. JS при этом идет выполнять код дальше, не дожидаясь ответа.

## 2. Очереди (Queues)

Когда Web API заканчивает свою работу (таймер дотикал или данные с сервера пришли), он не может просто вбросить коллбэк обратно в Call Stack (иначе код сломается). Он кладет его в специальную "комнату ожидания" — очередь. Их две:

### Microtask Queue (Очередь микрозадач)

- **VIP-очередь с наивысшим приоритетом.**
- **Что сюда попадает:** Коллбэки промисов (`.then`, `.catch`, `.finally`), `queueMicrotask`, `MutationObserver`.
- **Важное правило:** Event Loop не перейдет к макрозадачам и не будет перерисовывать экран (UI Render), пока очередь микрозадач не станет *абсолютно пустой*. Если микрозадача порождает новую микрозадачу, они будут выполняться бесконечно, блокируя страницу.

### Macrotask Queue (Очередь макрозадач / Task Queue)

- **Обычная очередь.**
- **Что сюда попадает:** `setTimeout`, `setInterval`, `setImmediate` (в Node.js), обработчики событий (клики, скроллы), I/O (сеть, жесткий диск).
- **Важное правило:** За один цикл (тик) Event Loop берет только **ОДНУ** макрозадачу, выполняет ее, а затем снова идет проверять микрозадачи.

## 3. Алгоритм работы Event Loop (Золотое правило)

Событийный цикл — это буквально бесконечный цикл (как `while(true)`), который постоянно проверяет: *"Стек пуст? Если пуст, что там в очередях?"*.

**Порядок выполнения (Тик):**

1. Выполнить весь синхронный код (пока Call Stack не опустеет).
2. Проверить **Microtask Queue**. Выполнить *все* микрозадачи до одной, пока очередь не опустеет.
3. *Отрисовать изменения на экране (UI Rendering - происходит примерно раз в 16мс).*
4. Проверить **Macrotask Queue**. Взять **одну** (самую старую) макрозадачу и закинуть ее в Call Stack.
5. Вернуться к шагу 1.
