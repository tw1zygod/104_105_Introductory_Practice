# Прототипы и Прототипное наследование

В JavaScript нет "классов" в традиционном понимании (как в Java или C#). Классы в JS — это синтаксический сахар над функциями и **прототипным наследованием**.

## 1. Главный секрет объектов (Скрытое свойство)

У каждого объекта в JavaScript есть скрытое внутреннее свойство `[[Prototype]]`.

Оно может быть либо равно `null`, либо ссылаться на **другой объект** (который и называется его "прототипом").

Когда мы пытаемся прочитать свойство или вызвать метод у объекта, а его там нет, движок JS не сдается. Он идет по ссылке `[[Prototype]]` и ищет это свойство в объекте-прототипе.

## 2. Битва терминов: `__proto__` vs `prototype`

Это самый частый вопрос на понимание. Эти два слова звучат похоже, но делают разные вещи:

### `__proto__` (Где искать родителя?)

- Это геттер/сеттер, через который мы (разработчики) можем получить доступ к скрытому `[[Prototype]]` **конкретного созданного объекта**.
- *Важно:* В современном коде использовать `__proto__` напрямую считается плохим тоном (он устарел). Вместо него используют `Object.getPrototypeOf(obj)`.

### 🔹 `prototype` (Чертеж для детей)

- Это обычное свойство, которое есть **только у функций** (за исключением стрелочных).
- Оно нужно только для одного: когда мы вызываем функцию с оператором `new` (создаем объект), движок берет объект, лежащий в свойстве `prototype` этой функции, и устанавливает его как `__proto__` для нового объекта.

```jsx
function User(name) {
  this.name = name;
}
User.prototype.sayHi = function() { console.log("Привет!"); };

const petya = new User("Петя");

// Движок сделал так: petya.__proto__ = User.prototype;
petya.sayHi(); // Петя идет в свой __proto__ и находит там sayHi
```

## 3. Цепочка прототипов (Prototype Chain)

Что произойдет, если свойства нет ни в самом объекте, ни в его прототипе? Движок пойдет в прототип прототипа. И так далее, пока не упрется в `null`.

**Пример с массивом:**

```jsx
const arr = [1, 2, 3]; // Это то же самое, что new Array(1, 2, 3)
```

1. У `arr` есть свои данные: `0: 1`, `1: 2`, `length: 3`. Но метода `map` там нет.
2. Мы пишем `arr.map()`.
3. JS ищет `map` в `arr`. Не находит.
4. JS идет в `arr.__proto__` (он ссылается на `Array.prototype`). О, здесь лежат все методы массивов: `map`, `filter`, `reduce`! Нашел, выполнил.
5. А если мы вызовем `arr.hasOwnProperty()`? Его нет даже в `Array.prototype`.
6. JS идет дальше: в `Array.prototype.__proto__` (он ссылается на `Object.prototype`). Находит там!
7. Дальше `Object.prototype.__proto__` равно `null`. Цепочка закончилась.

## 4. Зачем нам это для полифилов? (Наш `Array.prototype.myReduce`)

Почему на собеседованиях просят писать полифил именно так:

`Array.prototype.myReduce = function() { ... }`

Потому что, добавляя метод напрямую в "чертеж" `Array.prototype`, мы делаем так, что **абсолютно любой массив** в нашем коде (чей `__proto__` неизбежно ссылается на этот чертеж) мгновенно получает к нему доступ.

Внутри этой функции `this` будет указывать на тот массив, который стоит перед точкой при вызове `[1, 2].myReduce()`.
