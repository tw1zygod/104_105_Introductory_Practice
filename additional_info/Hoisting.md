# Hoisting (Поднятие) и Временная мертвая зона

Прежде чем выполнить код, движок JavaScript проходит по нему и "собирает" все объявления переменных и функций. Этот процесс называется **Hoisting (Поднятие)** — объявления как бы "поднимаются" на самый верх своей области видимости. Но `var`, `let` и `const` ведут себя при этом совершенно по-разному.

## Поведение `var`

Переменные, объявленные через `var`, поднимаются и **сразу инициализируются значением `undefined`**.

Поэтому мы можем обратиться к `var` до ее объявления в коде, и ошибки не будет.

```jsx
console.log(name); // Выведет: undefined (ошибки нет!)
var name = "Ozon";
console.log(name); // Выведет: "Ozon"
```

## Поведение `let` и `const` (TDZ)

Переменные `let` и `const` **тоже поднимаются** (движок знает об их существовании). Но, в отличие от `var`, они **не инициализируются**.

Период от начала блока кода до строчки, где переменная объявлена, называется **Temporal Dead Zone (TDZ — Временная мертвая зона)**. Попытка обратиться к переменной в этой зоне вызовет ошибку.

```jsx
console.log(age); // ReferenceError: Cannot access 'age' before initialization
let age = 25;
```

# 2. Контекст `this` и его коварная потеря

**Главное правило `this`:** В классических функциях (не стрелочных) `this` определяется **в момент вызова функции**, а не в момент ее создания. `this` — это объект, который стоит *перед точкой* при вызове метода.

## Как происходит потеря контекста в `setTimeout`

Представь объект с методом, который мы передаем в коллбэк (например, обработчик события или таймер):

```jsx
const user = {
  name: "Иван",
  greet() {
    console.log(`Привет, я ${this.name}`);
  }
};

// 1. Прямой вызов (все отлично):
user.greet(); // Выведет: "Привет, я Иван" (перед точкой стоит user)

// 2. Потеря контекста:
setTimeout(user.greet, 1000); // Выведет: "Привет, я undefined"
```

**Почему это произошло?**

В `setTimeout` мы передаем *только саму функцию* (ссылку на нее в памяти), отрывая ее от объекта `user`. Когда таймер срабатывает, функцию вызывает сам движок (среда). Так как перед вызовом нет "точки", `this` становится равен глобальному объекту `window` (или `undefined` в строгом режиме `"use strict"`). А в `window` нет свойства `name`.

## 3 способа решить проблему (Как привязать контекст)

**Способ 1: Стрелочная функция-обертка (Самый частый)**

Мы не передаем метод напрямую, а создаем новую стрелочную функцию. Когда она выполнится, она вызовет `user.greet()` обычным способом (с точкой).

```jsx
setTimeout(() => user.greet(), 1000); // "Привет, я Иван"
```

**Способ 2: Жесткая привязка через `.bind()`**

Метод `bind` создает *новую* функцию, намертво привязывая к ней контекст, который мы передали в скобках.

```jsx
setTimeout(user.greet.bind(user), 1000); // "Привет, я Иван"
```

**Способ 3: Сделать сам метод стрелочной функцией (в классах)**

Стрелочные функции вообще не имеют своего `this`. Они всегда берут `this` из внешнего лексического окружения (там, где были написаны).

```jsx
const user = {
  name: "Иван",
  greet: () => { console.log(`Привет, я ${this.name}`) } // Здесь this возьмется из window!
};
```
