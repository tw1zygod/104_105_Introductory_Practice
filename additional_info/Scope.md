# Область видимости (Scope) и Замыкания (Closures)

Чтобы понять замыкания, нужно сначала разобраться, как JavaScript хранит переменные под капотом. Для этого существует концепция **Лексического окружения**.

## Lexical Environment (Лексическое окружение)

**Что это:** Это скрытый внутренний объект движка JS, который создается каждый раз, когда вызывается функция, выполняется блок кода `{ ... }` или запускается весь скрипт.

У Лексического окружения есть **две части**:

1. **Environment Record (Запись окружения):** Это объект, в котором как свойства хранятся все локальные переменные (и значение `this`).
2. **Ссылка на внешнее окружение:** Ссылка на то лексическое окружение, внутри которого физически (в коде) была создана наша функция.

**Как работает поиск переменных (Scope Chain):**

Когда код хочет прочитать переменную, он сначала ищет ее во внутреннем Лексическом окружении. Если не находит — идет по ссылке во внешнее. И так далее, вплоть до глобального окружения (`window`). Если переменной нет и там — выкидывается ошибка `ReferenceError`.

## Замыкания (Closures)

**Главное определение:** Замыкание — это функция **вместе** со всеми внешними переменными, которые ей доступны.

В JavaScript **все функции изначально являются замыканиями** (исключение — функции, созданные через `new Function`).

### Как функция «запоминает» место своего рождения?

Секрет кроется в скрытом свойстве `[[Environment]]`.

Когда функция создается (именно создается, а не вызывается!), она навсегда записывает в свое свойство `[[Environment]]` ссылку на Лексическое окружение того места, где она была написана.

Даже если ты передашь эту функцию в `setTimeout` или вернешь ее из другой функции (вызовешь в совершенно другом месте), она всё равно будет искать переменные по своей ссылке `[[Environment]]`, то есть **там, где родилась, а не там, где выполняется**.

**Классический пример (Счетчик):**

```jsx
function createCounter() {
  let count = 0; // Переменная во внешнем лексическом окружении

  return function() {
    count++; // Функция находит count снаружи и меняет его
    return count;
  };
}

const counter1 = createCounter();
console.log(counter1()); // 1
console.log(counter1()); // 2
```

В этом примере внутренняя функция замкнула на себе переменную `count`. Когда `createCounter` отработала, ее Лексическое окружение не удалилось сборщиком мусора, потому что на него осталась ссылка внутри возвращенной функции `counter1`.
