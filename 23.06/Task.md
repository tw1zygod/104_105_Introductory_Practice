# Практическое задание: Проектирование ядра данных сайта (ЛР7)

## Цель задания

Разработать сложную многомерную базу данных для вашего сайта-визитки. Научиться извлекать, фильтровать и трансформировать информацию с помощью методов `map`, `filter`, `forEach`, а также использовать продвинутые коллекции `Set` и `Map`.

## Техническое задание

Работа выполняется в файле `script.js` (или в консоли браузера). Результаты каждого шага необходимо выводить с помощью `console.log()`.

### Шаг 1: Проектирование архитектуры (Создание БД)

Создайте глобальный объект `portfolioData`, который будет хранить всю информацию о вашем сайте. Внутри этого объекта должно быть три свойства:

1. **`profile`** — объект с информацией о вас (ключи: `name`, `profession`, `age`).
2. **`projects`** — массив объектов. В массиве должно быть **не менее 5 проектов**. Каждый проект обязан иметь ключи: `id`, `title`, `category` (например, "Веб", "Дизайн", "Бекенд"), `likes` (число) и `technologies` (массив строк, например `["HTML", "CSS"]`). *Сделайте так, чтобы категории и технологии в разных проектах повторялись.*
3. **`preferences`** — коллекция `Map`, которая хранит настройки сайта. Добавьте туда два ключа: строку `"theme"` (значение `"dark"`) и строку `"language"` (значение `"ru"`).

### Шаг 2: Анализ профиля (Методы объектов)

Вашему скрипту нужно быстро понять, какие вообще поля есть в профиле пользователя, чтобы потом отрисовать их в интерфейсе.

1. Используя встроенный класс `Object`, получите массив всех **ключей** из объекта `profile`.
2. Выведите этот массив ключей в консоль.

### Шаг 3: Выборка лучших работ (Методы массивов `filter` и `map`)

На главной странице сайта мы хотим показывать только самые залайканные работы.

1. Обратитесь к массиву `projects` внутри вашей базы данных.
2. Используя метод `filter()`, отфильтруйте массив так, чтобы остались только проекты, у которых строго **больше 100 лайков**.
3. Сразу после этого (или отдельным шагом) используйте метод `map()`, чтобы превратить отфильтрованные объекты в массив обычных строк.
* Формат строки должен быть строго таким: `"Проект: [Название] из категории [Категория]"`.
4. Выведите полученный массив строк в консоль.

### Шаг 4: Подсчет статистики (Циклы)

Для панели администратора нужно посчитать общее количество лайков, собранных всеми вашими проектами вместе.

1. Создайте переменную для хранения суммы.
2. Пройдитесь по всем проектам и сложите их лайки в эту переменную.
3. Выведите итог в консоль в формате: `"Общее число лайков: [сумма]"`.

### Шаг 5: Облако тегов (Коллекция `Set`)

Для создания фильтра на сайте вам нужен список всех технологий, которые вы когда-либо использовали. Но в массивах `technologies` они повторяются!

1. Создайте пустую коллекцию `Set`.
2. Пройдитесь по всем проектам, загляните в каждый массив `technologies` и добавьте каждую технологию в ваш `Set`.
3. Убедитесь, что в итоговом объекте `Set` нет ни одного дубликата (каждая технология упоминается только один раз), и выведите его в консоль.
